pyfuncs_to_chars

any character as python builtins — no numbers, no strings, no operators

how it works

Start from not() → True. Through str(), min(), ord() and friends, we extract integers from Python's own string representations. These are our base anchors — about 40 numbers from 0 to 125.

The key insight: len(str(list(bytes(n)))) = exactly 3n.

Combined with max(range(n)) = n−1, any number can be built in base 3. Recursively build ceil(n/3), triple it, then decrement 0–2 times:

97 = 3 × 33 − 2 = 3 × (3 × 11) − 2 = 3 × (3 × len(str(frozenset()))) − 2

Then wrap in chr(). Max depth for any Unicode character: ~114 parens out of Python's 200 limit.

The optimizer tries faster strategies — 4n+3, 5n+5, zip chains (6n, 9n), and the ascii exponential ((2^k+3)n) — to find shorter expressions.
